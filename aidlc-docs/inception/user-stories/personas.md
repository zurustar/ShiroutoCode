## ペルソナ P1: 開発者「タクミ」（Primary / 唯一のペルソナ）

- **役割**: ソフトウェアエンジニア。日常的にコードを書き、ターミナルでの作業に慣れている。
- **技術習熟度**: 中〜上級。CLIやGitに慣れている。ターミナルからツールを使うことに抵抗がない。
- **動機 / ゴール**:
  - AIにコーディング作業（複数ファイルの変更、リファクタ、ボイラープレート生成、ビルド/テスト実行）を任せて生産性を上げたい。
  - クラウドのAIサービスにコードを送りたくない。**手元のLM Studioのローカルモデル**で完結させたい（機密・オフライン・コスト）。
- **行動特性**:
  - 指示を出したら基本は**自動で進めて**ほしい。逐一承認を求められるのは煩わしい。
  - ただし、システムを壊す・外部を攻撃するような操作は**絶対に避けて**ほしい。
  - 何が実行されたかは後から追える（可視化・履歴）ことを重視する。
- **利用環境**:
  - **複数のPCを行き来して作業する**（リポジトリ経由で状態を持ち運ぶ）。
  - LM Studio をローカルで起動し、OpenAI互換APIを提供している。
- **不満 / ペインポイント**:
  - クラウドAIはプライバシー・コスト・オフライン不可が不安。
  - 既存ツールは承認が多すぎる、または逆に無制限で危険。
  - LM Studio未起動など接続トラブル時に原因が分かりにくいと困る。
- **成功の定義**:
  - 自然言語の指示一つで、複数ファイルにまたがる変更が安全に完遂され、結果を確認できる。
